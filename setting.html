🚀 DB 자동 구성

버튼 한 번으로 테이블 생성 · RLS 설정 · Realtime 활성화까지 자동 처리

💡 이미 테이블이 존재하면 건드리지 않고 스킵합니다. 중복 생성 걱정 없이 실행하세요.
1️⃣
edu_sessions 테이블
학습자 로그인 세션, 온라인 상태, 포인트, 현재 카드 위치 저장
대기 중
2️⃣
edu_progress 테이블

대기 중
3️⃣
edu_chat 테이블
페이지별 실시간 채팅 메시지, 파일/이미지 첨부 정보 저장
대기 중
4️⃣
RLS 정책 확인
Row Level Security 활성화 및 익명(anon) 접근 정책 설정 확인
대기 중
5️⃣
Storage 버킷 (edu-chat-files)
채팅 파일/이미지 업로드용 Public 버킷 생성
대기 중
6️⃣
password 컬럼 확인
edu_sessions 테이블에 학습자 개인 비밀번호 저장 컬럼 존재 여부 확인
대기 중
✅ Realtime: Database → Publications → supabase_realtime에서 edu_chat·edu_sessions 토글 ON
✅ Storage Policy: SQL Editor에서 3개 Policy 실행 (storage.objects에 anon SELECT/INSERT/DELETE)
✅ Storage 버킷: Dashboard → Storage → New Bucket → edu-chat-files → Public ON

💥 테이블별 전체 삭제

선택한 테이블의 모든 행을 삭제합니다. 테이블 구조는 유지됩니다.

⚡ 완전 초기화 (전체 삭제)

세션 + 진행률 + 채팅 3개 테이블 전체를 한 번에 비웁니다. 절대 되돌릴 수 없습니다.

🔐 비밀번호 관리

학습자별 비밀번호는 DB(edu_sessions)에 저장됩니다

💡 동작 방식
• 처음 닉네임+비밀번호 입력 → DB에 계정 생성
• 같은 닉네임으로 재접속 시 → 저장된 비밀번호와 일치해야 입장
• 다른 사람이 같은 닉네임+다른 비밀번호 → ❌ 차단
• 동시 접속 (5분 이내 활성) → ❌ 차단

🔑 특정 학습자 비밀번호 초기화

학습자가 비밀번호를 잊었을 때 새 비밀번호로 변경해줍니다.

📋 전체 학습자 비밀번호 목록

등록된 모든 학습자의 닉네임과 비밀번호를 조회합니다.

⚡ 강제 로그아웃

특정 학습자의 세션을 즉시 만료시킵니다 (last_seen을 과거로 설정)

세션을 만료시키면 해당 학습자는 5분 내로 자동 로그아웃되거나 다음 접속 시 재로그인이 필요합니다.
